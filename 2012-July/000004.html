On Wed, Jul 11, 2012 at 2:16 PM, Haines Brown <haines at histomat.net> wrote:
> Setting up wifi on a new installation of debian squeeze on a Thinkpad
> X61s. I have wpasupplicant installed, but not being used because WPA
> is not yet enabled in my modem. I am not using any kind of network
> manager such as wicd.
>
> The wifi connection worked fine until I installed and started
> wifi-radar.

  How were you able to get a wifi connection without wicd or wifi-radar?

> At first wifi-radar seemed to work. It scanned, I created
> a profile for my home AP. Except it didn't report signal strenghs. But
> in fiddling to configure the profile (adding my AP's inet address,
> which made no sense to me to do), it hung, and since I couldn't
> bring the wlan0 interface up.
>
>   # /etc/init.d/networking restart
>   ...
>   Listening on LPF/wlan0/00:1f:3b:03:26:ab
>   Sending on  LPF/wlan0/00:1f:3b:03:26:ab
>   Sending on Socket/fallback
>   DHCPRELEASE on wlan0 to 192.168.1.254 port 67
>   DHCPDISCOVER on wlan0 to 255.255.255.255 port 67 interval 3
>   ...
>   No DHCOFFERS received
>   No working leases in persistent database - sleeping
>   done
>
> I try:
>
>   # iwlist scan
>   ...
>   wlan0  Interface doesn't support scanning: Device or resource busy
>
> I thought at first wifi-radar had somehow grabbed the interface and
> wouldn't go. However, when I kill wifi-radar and try to start it, it
> no longer starts, and I get:
>
>   # wifi-radar
>   Traceback (most recent call last)
>     File "/usr/sbin/wifi-radar",
>         line 2915, in <module> confFile.read()
>     File "/usr/sbin/wifi-radar",
>         line 2800, in read self.set_bool_opt( ap + 'known', True)
>     File "/usr/sbin/wifi-radar",
>         line 2713, in set_bool_opt self.set_opt(option_path, repr(value))
>     File "/usr/sbin/wifi-radar",
>         line 2762, in set_opt (section, option) = option_path.split{'.')
>
> When I reboot, wifi is back working, including # iwlist scan. But
> wifi-radar seems broken, and I get the same errors as above.
>

  This error comes from a broken configuration file (likely
/etc/wifi-radar/wifi-radar.conf or /etc/wifi-radar.conf).  It probably
was corrupted during the freeze when configuring.  Delete the
configuration file and I believe WiFi Radar will start up.

  Please let the list know if this works or not.
